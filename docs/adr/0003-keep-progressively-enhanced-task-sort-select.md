# Keep the progressively enhanced Task Sort select

We will keep the customized Task Sort select that uses `appearance: base-select` as a progressive enhancement instead of replacing it with a plain native select. The select's baseline control height must be set on the `<select>` itself so browsers with partial support, including Safari, still render a usable 44px-plus control. The custom select children should be authored as JSX, not injected with `dangerouslySetInnerHTML`. Future responsive or accessibility passes should preserve this enhancement unless it breaks native fallback behavior, keyboard access, or Task Sort semantics.
